# Collaboration Guidelines
# Branch Naming Convention

Branches must follow a consistent naming scheme based on the type of work being done:
Feature development
feature/<name-of-feature>
Bug fixes or corrections
fix/<name-of-fix>
Branch names should be short, descriptive, and written in lowercase with hyphens separating words.

# Commit Messages
Use clear and descriptive commit messages for proper versioning and traceability
Keep messages concise but meaningful
Recommended format:
<type>: <short description>
Examples:
feat: add user authentication endpoint
fix: resolve token validation issue
# Merging Branches
A branch should be reviewed before merging
At least two team members should approve the changes when possible
Ensure the code is tested and functional before requesting a merge
# Pushing and Synchronization:
Always pull the latest changes before starting new work
Always pull before pushing changes to avoid conflicts
Keep your branch up to date with the base branch (typically develop)
